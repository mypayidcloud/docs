## Prerequisites

You'll need an active account on the PayID Platform. Every API call requires either a **Bearer access token** (issued via OAuth) or an **API Key**.

---

## Step 1: Create an Organization

An organization is the root container for everything in PayID — API keys, OAuth clients, members, and analytics all belong to an org.

```bash
curl -X POST https://api.payid.aevr.space/api/v1/organizations \
  -H "Authorization: Bearer <your_access_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Acme Corp",
    "slug": "acme",
    "description": "My first organization"
  }'
```

A successful response returns the organization object including its `id`, which you'll use for all subsequent requests.

---

## Step 2: Generate an API Key

API keys allow you to authenticate API requests without going through OAuth. Use them for server-to-server calls.

```bash
curl -X POST https://api.payid.aevr.space/api/v1/organizations/<orgId>/api-keys \
  -H "Authorization: Bearer <your_access_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My Server Key",
    "permissions": ["org:read", "member:read"]
  }'
```

<Warning>
  The full API key value is only returned **once** at creation. Store it securely — you cannot retrieve it again.
</Warning>

---

## Step 3: Make Your First API Request

Use your new API Key to authenticate requests using the `Authorization` header with the `ApiKey` scheme:

```bash
curl https://api.payid.aevr.space/api/v1/organizations/<orgId> \
  -H "Authorization: ApiKey <your_api_key>"
```

A successful response will return your organization's details.
