title: "Organizations"

---

Organizations are the top-level entity in PayID. Every resource — API keys, OAuth clients, settings, analytics, and member roles — belongs to an organization.

---

## Creating an Organization

```bash
curl -X POST https://api.payid.cloud/api/v1/organizations \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Acme Corp",
    "slug": "acme",
    "description": "Our main organization"
  }'
```

The `slug` must be unique across the platform and is used in URLs and identifiers.

---

## Roles

Every org member has one of the following built-in roles:

| Role | Capabilities |
|---|---|
| `owner` | Full control, can transfer ownership |
| `admin` | Manage members, settings, API keys |
| `developer` | Manage OAuth clients, view analytics |
| `viewer` | Read-only access |

## Managing Members

### Inviting a Member

```bash
curl -X POST https://api.payid.cloud/api/v1/organizations/<orgId>/members/invite \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "dev@example.com",
    "role": "developer"
  }'
```

The invited user receives an email. They accept via a link which calls:

```bash
POST /api/v1/organizations/<orgId>/members/invitations/<invitationId>/accept
```

### Updating a Member's Role

```bash
curl -X PATCH https://api.payid.cloud/api/v1/organizations/<orgId>/members/<memberId> \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{ "role": "admin" }'
```

### Removing a Member

```bash
curl -X DELETE https://api.payid.cloud/api/v1/organizations/<orgId>/members/<memberId> \
  -H "Authorization: Bearer <access_token>"
```

## Organization Lifecycle

### Suspending an Organization

Suspending prevents all API calls for the organization's clients and API keys until reactivated.

```bash
curl -X POST https://api.payid.cloud/api/v1/organizations/<orgId>/suspend \
  -H "Authorization: Bearer <access_token>"
```

### Transferring Ownership

```bash
curl -X POST https://api.payid.cloud/api/v1/organizations/<orgId>/transfer-ownership \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{ "newOwnerId": "<userId>" }'
```

<Warning>
  Ownership transfer is irreversible without the new owner transferring it back.
</Warning>

### Exporting Data

Export all organization data as a JSON archive:

```bash
curl https://api.payid.cloud/api/v1/organizations/<orgId>/export \
  -H "Authorization: Bearer <access_token>"
```
