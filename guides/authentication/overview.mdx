The PayID Platform supports two primary authentication mechanisms for API requests:

| Method | Header Format | Best For |
|---|---|---|
| **Bearer JWT** | `Authorization: Bearer <token>` | User-scoped actions, OAuth flows |
| **API Key** | `Authorization: ApiKey <key>` | Server-to-server calls |


## Bearer JWT (Access Tokens)

Bearer tokens are JWTs issued by the PayID OAuth authorization server. They carry the user's identity, scopes, and organization context.

### Obtaining a Token

Use the token endpoint with the `client_credentials` or `authorization_code` grant:

```bash
curl -X POST https://api.payid.aevr.space/oauth/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=client_credentials&client_id=<id>&client_secret=<secret>&scope=org:read"
```

```json
{
  "access_token": "eyJhbGci...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "scope": "org:read"
}
```

### Using the Token

Pass the access token in every API request via the `Authorization` header:

```bash
curl https://api.payid.aevr.space/api/v1/organizations/<orgId> \
  -H "Authorization: Bearer eyJhbGci..."
```

### Token Expiry & Refresh

Access tokens are short-lived (default: 1 hour). Use the `refresh_token` grant to obtain a new access token without re-authenticating:

```bash
curl -X POST https://api.payid.aevr.space/oauth/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=refresh_token&refresh_token=<token>&client_id=<id>"
```

## API Keys

API Keys are long-lived credentials scoped to an organization. They are ideal for backend services that need to make API calls without an interactive OAuth flow.

### Creating an API Key

```bash
curl -X POST https://api.payid.aevr.space/api/v1/organizations/<orgId>/api-keys \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Production Server",
    "permissions": ["org:read", "member:read", "apikey:read"]
  }'
```

### Using an API Key

```bash
curl https://api.payid.aevr.space/api/v1/organizations/<orgId> \
  -H "Authorization: ApiKey pk_live_abc123..."
```

<Warning>
  API Keys are organization-scoped. They cannot perform user-specific actions like token issuance or consent management. Use OAuth access tokens for those operations.
</Warning>

## Identifying Yourself

If you have an active access token and want to introspect it or get the associated user claims, use:

```bash
# Introspect a token (requires client credentials)
curl -X POST https://api.payid.aevr.space/oauth/introspect \
  -d "token=<access_token>&client_id=<id>&client_secret=<secret>"

# Get user claims from an access token
curl https://api.payid.aevr.space/oauth/userinfo \
  -H "Authorization: Bearer <access_token>"
```
